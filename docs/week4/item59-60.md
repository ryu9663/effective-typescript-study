# Item 59-60

## Item 59 타입스크립트 도입 전에 @ts-check와 JSDoc으로 시험해보기

### @ts-check

```js
// @ts-check
const person = { first: "Grace", last: "Hopper" };
2 * person.first; // string에 곱셈 → 타입 오류
```

### .d.ts로 빈약한 타입 보강

- 외부 라이브러리를 사용하는데 타입이 빈약할때 (예를들어 <script/> 내의 타입)

  카카오맵의 경우에 아래와 같이 타입 보강

  ```ts
  // declaration.d.ts

  interface Window {
    kakao: any;
  }
  ```

### 라이브러리의 @types 설치하기

JQuery같은건 '#'을 인식못하니까 @types/jquery 다운받기

```js
// @ts-check
$("#graph").style({ width: "100px" });
```

### JSDoc

JS에서 @ts-check 지시자는 매우 느슨한 타입 체크를 한다.

이때 @param, @returns, @type 같은 JSDoc 주석을 붙힌 후 @ts-check를 붙히면 타입 검사가 더 좋아진다.

## Item 60 allowJs로 타입스크립트와 자바스크립트 같이 사용하기

대규모 JS 코드베이스를 TS로 전환할 경우 allowJs 사용하기

allowJs 컴파일러 옵션을 사용하면 TS와 JS를 서로 임포트 할 수 있다.

- TS와 JS를 공존하게 한다.
- TS와 JS를 서로 임포트하게 해준다.
- 제대로 된 마이그레이션을 시작하기 위해 반드시 필요하다.
